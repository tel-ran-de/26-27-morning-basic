HTML(HyperText Murkup Language - "язык гипертекстовой разметки") - самый базовый строительный блок ВЕБА. Он определяет содержание и структуру веб-контента. Другие технологии обычно используются для описания внешнего вида/представления (CSS) или функциональности/поведения (JavaScript) веб/страницы.

Под гипертекстом ("hypertext") понимаются ссылки, которые соединяют веб-страницы друг с другом в пределах одного ве-сайта или между веб-сайтами. Ссылки являются фундаментальным аспектом Веба. Загружая контент в Интернет и связывая его со страницами, созданными другими людьми, вы сами становитесь участником Всемирной паутины.

HTML использует разметку ("markup") для отображения текста, изображеинй и другого контента в веб-браузере. HTML -елемент выделяется изпрочего текста с помощью "тегов", которые состоят из имени элемента, окружённого. Имя элемента внутри тега не чувствительно к регистру; ономожет быть написано любым способом.

Введение в HTML

По сути, HTML - довольно простой язык, состоящий из элементов, которые могут быть применены к частям текста, чтобы придавать им различные значения (абзац, список, таблица), разделять документ на логические секции (шапка, контент,меню навигации) и добавлять фото или видео на вашу страницу.

HTML- таблицы

Структурирование табличных данных является в HTML самой обычной задачей, для этой цели есть множество эелементов и атрибутов.

Таблица - это структурированный набор данных, состоящий из строк и столбцов (табличные данные). Таблицы позволяют быстро и легко посмотреть значения, показывающие некоторую взаимосвязь между различными типами данных. Смысл таблицы в её жёсткости. Информацию легко интерпретировать, визуально сопоставляя заголовки строк и столбцов.

HTML-формы

Формы являются очень важной частью Веба — они предоставляют большую часть функциональности, необходимого для взаимодействия с веб-сайтами, например регистрация и вход в систему, отправка отзывов, покупка товаров, и многое другое.

Css - селекторы

В CSS селекторы используются ждя стилизации HTML элементов на веб-странице. Существует широкий выбор селекторов, позволяющий максимально точно отбирать элементы для стилизации. Селекторы говорят браузеру к какому элементу HTML нужно приментить те или иные свойства CSS, заданные внутри блока объявлеиня стиля. Селектор определяет, к какому элементу применять то или иное правило CSS. Важно то, что не существует таких селекторов, которые позволили бы выбираь родителя (то, что содержить определеннный контейнер) или соседа родителя или потомков соседа родителя.

Базовый Универсальный Селектор выбирает все элементы. (по желанию он может быть ограничен определённым пространством имён или относиться ко всему пространсву имён.)

Базовый Селектор по типу элемента выбирает тип элементов, к которым будет применяться правило.

Базовый Селекторпо классу выбирает элементы, основываясь на значении их атрибута class.

Базовый селектор по индентификатору выбирает элементы, основываясь на значении их id атрибута, который может использоваться только один раз для одного и того же элемента в HTML документе.

Базовый Селектор по атрибуту выбирает все элименты, имеющие данный атрибут или просто выбирает атрибут с определённым значением.

Дочерний селектор выбирает через комбинатор " > " только те элементы, которые являются дочерними по отношению к указанному элементу. Синтаксис: A > B

Элементы HTML являются обычно "строчными" или "блочными". Строчный элемент занимает только пространство, ограниченное тегами, которые его определяют. Блочный элемент занимает все пространство своего родительского элемента (контейнера), тем самым создавая "блок".

В HTML различные типы ссылок могут использоваться для установления и определения связи между двумя документами. Элементы позволяют вам воспроизводить аудио и видео. Эти элементы предоставляют браузерную альтернативу аналогичным возможностям, которые есть в Adobe Flash и других плагинах. Каждый элемент HTML принадлежит некоторому количеству категорий контента, которые объединяют элементы с общим набором характеристик.

Вёрстка слоями сложнее, чем таблицами, поскольку требует от разработчика глубоких знаний спецификации CSS. Браузеры содержат разные ошибки при интерпретации стилевых свойств, поэтому следует знать особенности поведения основных браузеров, их ошибки и уметь обходить их.
